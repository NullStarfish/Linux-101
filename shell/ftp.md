在 Linux 环境下，`ftp` 命令的操作通常分为两类：**启动时的命令行选项 (Options)** 和 **连接后的交互式命令 (Internal Commands)**。

由于传统的 `ftp` 协议不加密（明文传输），现在更多人使用 `sftp` 或 `lftp`，但基础的 `ftp` 命令在内网环境或旧系统中依然常用。

---

### 1. 启动时的常用命令行选项 (Options)
这些是在终端输入 `ftp [options] [hostname]` 时使用的参数：

| 选项       | 说明                      | 常用场景                                       |
| :------- | :---------------------- | :----------------------------------------- |
| **`-p`** | **被动模式 (Passive mode)** | **最常用**。解决防火墙或 NAT 导致的连接后无法列出目录或传输文件的问题。   |
| **`-i`** | **关闭交互提示**              | 在使用 `mget` 或 `mput` 传输多个文件时，不再询问确认，直接全部传输。 |
| **`-n`** | **禁用自动登录**              | 启动时不询问用户名密码。常用于脚本中配合 `user` 命令手动输入或自动化登录。  |
| **`-v`** | **详细模式 (Verbose)**      | 显示服务器返回的所有响应信息，方便查看传输状态。                   |
| **`-d`** | **调试模式 (Debug)**        | 显示更详细的底层指令交互，用于排查连接故障。                     |
| **`-g`** | **禁用通配符 (Globbing)**    | 禁止对文件名中的特殊字符（如 `*`、`?`）进行扩展。               |
| **`-A`** | **匿名登录**                | 尝试以匿名身份直接登录（如果服务器允许）。                      |

**用法示例：**
```bash
# 以被动模式连接，并关闭多文件传输时的提示
ftp -p -i 192.168.1.100
```

---

### 2. 连接后的常用交互命令 (Internal Commands)
登录成功后，你会进入 `ftp>` 提示符，此时常用的操作命令如下：

#### 文件传输
*   **`get [remote-file] [local-file]`**：下载单个文件。
*   **`put [local-file] [remote-file]`**：上传单个文件。
*   **`mget [wildcard]`**：批量下载（如 `mget *.txt`）。
*   **`mput [wildcard]`**：批量上传（如 `mput *.jpg`）。
*   **`reget`**：断点续传下载（如果下载中断，从上次的位置继续）。

#### 目录与路径切换
*   **`ls / dir`**：列出远程目录下的文件。
*   **`cd [directory]`**：切换远程目录。
*   **`lcd [directory]`**：**Local CD**，切换**本地**所在的目录（非常实用）。
*   **`pwd`**：显示远程当前工作目录。
*   **`mkdir` / `rmdir`**：创建或删除远程目录。

#### 传输模式设置
*   **`binary` (或 `bin`)**：**二进制模式**。用于传输图片、压缩包、可执行文件等（**推荐默认开启**）。
*   **`ascii`**：**文本模式**。用于传输纯文本、脚本，会自动转换换行符（可能会损坏非文本文件）。

#### 其他
*   **`prompt`**：切换交互模式开关（如果在启动时没加 `-i`，可以在这里输入一次 `prompt` 来关闭询问）。
*   **`hash`**：每传输 1024 字节打印一个 `#` 号，用于观察大文件传输进度。
*   **`bye` / `quit`**：退出 FTP 会话。
*   **`!`**：临时切换回本地 Shell，输入 `exit` 返回 FTP。

---

### 3. 进阶建议：使用 lftp
如果你觉得原生 `ftp` 命令不好用（比如不支持 Tab 补全、不支持递归下载目录），建议使用 **`lftp`**，它是 Linux 下最强大的 FTP 客户端。

*   **支持递归下载整个文件夹**：`mirror remote_dir local_dir`
*   **支持多线程**：`mirror --parallel=5`
*   **支持 SFTP/HTTP 等多种协议**。

**安装：** `sudo apt install lftp` 或 `sudo yum install lftp`
**用法示例：**
```bash
lftp -u username,password -e "mirror /remote/path /local/path; quit" 192.168.1.100
```

### 总结
对于普通 `ftp` 命令，记住 **`-p`**（被动模式）和 **`-i`**（取消确认）以及内部的 **`lcd`**（本地路径切换）和 **`bin`**（二进制传输）这几个关键点，就能应对 90% 的场景。

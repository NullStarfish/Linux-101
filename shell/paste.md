`paste` 命令是 Linux 中一个简单但非常有用的工具，它主要用于**将多个文件的行并行（水平）合并**。

如果说 `cat` 是垂直合并文件（一个接一个），那么 `paste` 就是水平合并文件（一行对一行）。

---

### 一、 核心选项汇总

| 选项       | 长选项                 | 说明                             |
| :------- | :------------------ | :----------------------------- |
| **`-d`** | `--delimiters`      | **指定分隔符**。默认使用 **Tab (制表符)**。  |
| **`-s`** | `--serial`          | **串行合并**。将一个文件的所有行合并成一行（行列转置）。 |
| **`-z`** | `--zero-terminated` | 以 NUL 字符而不是换行符作为结束符。           |
| **`-`**  | (匿名参数)              | 表示从 **标准输入 (stdin)** 读取数据。     |
|          |                     |                                |

---

### 二、 常见用法示例

假设有两个测试文件：
`file1.txt`:
```text
Apple
Banana
Orange
```
`file2.txt`:
```text
$1.0
$0.5
$0.8
```

#### 1. 基础并行合并 (默认 Tab 分隔)
这是最常见的用法，将两个文件的对应行拼在一起。
```bash
paste file1.txt file2.txt
# 输出:
# Apple   $1.0
# Banana  $0.5
# Orange  $0.8
```

#### 2. 指定自定义分隔符 (`-d`)
可以使用冒号、逗号或其他字符作为分隔符。
```bash
paste -d ':' file1.txt file2.txt
# 输出: Apple:$1.0 (以此类推)

# 甚至可以使用循环分隔符（如果合并多个文件）
paste -d ',|' file1 file2 file3
# 第一和第二列用 , 分隔，第二和第三列用 | 分隔
```

#### 3. 将一个文件的多行转为一行 (`-s`)
即“行列转置”，将纵向的列表变成横向的行。
```bash
paste -s file1.txt
# 输出: Apple   Banana  Orange
```

#### 4. 从标准输入读取 (`-`)
`paste` 可以通过 `-` 符号结合管道使用。
*   **示例：将一个列表每两个合并为一行**
    ```bash
    cat file1.txt | paste - -
    # 输出:
    # Apple   Banana
    # Orange
    ```
    *解释：两个 `-` 代表从标准输入读取两次，分别放在第一列和第二列。*

#### 5. 合并并保存到新文件
```bash
paste file1.txt file2.txt > result.csv
```

---

### 三、 实战技巧：结合其他命令

#### 1. 制作一个简单的 CSV 文件
```bash
paste -d ',' names.txt ages.txt jobs.txt > info.csv
```

#### 2. 快速生成序列对
如果你想把 1-10 的数字两两配对：
```bash
seq 10 | paste - -
# 输出:
# 1  2
# 3  4 ...
```

### 四、 paste vs join

虽然这两个命令都能合并文件，但有本质区别：

*   **`paste`**: 盲目合并。它不关心内容，只是简单地把第一文件的第一行和第二文件的第一行拼在一起。
*   **`join`**: 逻辑合并。它类似于 SQL 的 JOIN，需要两个文件有**共同的关联字段**（Key）才能进行合并。

**总结：**
*   如果你只是想把两列数据并排拼起来，用 **`paste`**。
*   如果你需要按某个 ID 匹配数据，用 **`join`**。
*   如果你需要处理复杂的列逻辑，用 **`awk`**。

`seq` (Sequence) 是 Linux 中一个非常实用的小工具，用于生成一个**数字序列**。它常用于 Shell 脚本中的循环、生成测试数据或批量创建文件。

### 一、 基础语法

`seq` 的参数顺序是：`[选项] [起始值] [步长] 结束值`

```bash
seq [选项] [开始] [步长] [结束]
```
*   如果只给一个数字，则默认从 1 开始，步长为 1，到该数字结束。
*   如果给两个数字，则表示 `开始` 和 `结束`。
*   如果给三个数字，则中间的是 `步长`。

---

### 二、 常用选项汇总

| 选项 | 长选项 | 说明 |
| :--- | :--- | :--- |
| **`-s`** | `--separator` | **指定分隔符**。默认为换行符 (`\n`)。 |
| **`-w`** | `--equal-width` | **等宽显示**。自动在数字前补 0，使所有数字长度一致。 |
| **`-f`** | `--format` | **格式化输出**。使用 printf 风格的格式（如 `%03g`）。 |

---

### 三、 常见用法示例

#### 1. 基础序列
*   **生成 1 到 5：**
    ```bash
    seq 5
    ```
*   **生成 5 到 10：**
    ```bash
    seq 5 10
    ```
*   **生成 1 到 10 之间的奇数（步长为 2）：**
    ```bash
    seq 1 2 10
    # 输出: 1, 3, 5, 7, 9
    ```

#### 2. 自定义分隔符 (`-s`)
默认是换行，你可以改为逗号、空格或横线：
```bash
seq -s " " 1 5
# 输出: 1 2 3 4 5

seq -s ", " 1 5
# 输出: 1, 2, 3, 4, 5
```

#### 3. 等宽补零 (`-w`)
在处理文件名（如 `img01.jpg`, `img02.jpg`）时非常有用：
```bash
seq -w 8 12
# 输出: 08, 09, 10, 11, 12
```

#### 4. 格式化输出 (`-f`)
使用 `%g`（默认）、`%e`（科学计数法）或 `%f`（浮点数）：
*   **生成带前缀的编号：**
    ```bash
    seq -f "User%03g" 1 3
    # 输出: User001, User002, User003
    ```
*   **生成两位小数：**
    ```bash
    seq -f "%.2f" 1 0.5 2
    # 输出: 1.00, 1.50, 2.00
    ```

#### 5. 倒序生成
只要步长设置为负数即可：
```bash
seq 5 -1 1
# 输出: 5, 4, 3, 2, 1
```

### 四、 实战场景

#### 1. 在 Shell 循环中使用 (最常见)
如果你想执行某个操作 10 次：
```bash
for i in $(seq 1 10); do
    echo "正在处理第 $i 项..."
done
```

#### 2. 批量创建目录或文件
快速创建 `dir_1` 到 `dir_20`：
```bash
mkdir $(seq -f "dir_%g" 1 20)
```

#### 3. 构造计算表达式
结合 `paste` 命令进行求和计算：
```bash
# 生成 1+2+3+4+5 并交给 bc 计算
seq -s "+" 5 | bc
# 输出: 15
```

### 五、 补充：`seq` vs `{start..end}`
在 Bash 中，你可能见过 `{1..5}` 这种写法：
*   **`{1..5}` (Brace Expansion):** 是 Bash 的内置特性。速度极快，但**不支持变量**（例如 `{$a..$b}` 是不行的）。
*   **`seq`:** 是一个外部命令。速度稍慢一点，但**支持变量传入**，功能更丰富（如格式化、浮点数步长）。

**结论：** 简单的固定循环用 `{1..n}`，需要灵活控制或在脚本里传入参数时用 `seq`。
